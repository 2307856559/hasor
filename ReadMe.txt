项目名：More

说明:
	

定义说明：
	项目名：项目的名称或者代号。
	版本号：项目的开发版本，例v1.0
	任务号：YYYYMMDD<_序号或代号>

目录说明:
	/Lost develop			(超前开发)
		(项目名)_(版本号)
		如果对项目有超前性功能开发可以在该目录中创建一个版本分支。

	/Release branches 		(发布版分支)
		(项目名)_(版本号)
		该目录中放置的是预期的发布版本项目。该目录中的最新版本是正在开发中的版本，有可能还没有开发完成。

	/Releases			(正式发布)
		(项目名)_(版本号)
		当开发进入到一个可以发布的版本时从发布版本中可以辅助一个项目版本到该目录中。该目录中的项目可能存在bug。一旦修复Bug将重新发布一个发布版到当前目录。

	/Resources			(资源)
		/lib 依赖的包。
		项目所依赖的资源，编译时依赖的资源。
	
	/Tasks				(开发任务)
		(任务号)-(项目名)_(版本号)
		如果涉及多人开发该目录可以用于存放某些可以分离出去的临时任务进行开发。

	/Trunk				(内核主干)
		/src
		/doc
		/db
		
	/Tags				(标记)
		(项目名)_(版本号)_标记摘要
		如果某些版本需要明确记住它但是这个版本还不具备发布的成熟度那么可以将这个版本拷贝到这个目录中以记住它，
