字节码生成工具，该工具可以在已有类型上附加接口实现，还可以对类对象提供AOP的支持，ClassEngine提供了两种工作模式。
介绍：
classcode只有1~2千行代码它不像CgLib拥有者上百个类去实现显着很臃肿。classcode内核使用的是ASM3.2框架内核。ASM是目前最 轻量化的字节码操作框架。它的内核也只有20个类因此classcode的核心选购也是非常优秀的。大名顶顶的Hibernate、Spring就使用的ASM作为字节码内核。 当然classcode不会拥有Spring那样丰富的功能，classcode也不会取代Spring。它只是一个字节码工具使用classcode可以取代CgLib。 性能方面会有相应的测试结果公布。
功能：
一、根据基类生成一个新的子类，并且在生成时候可以指定新类附加实现某些接口。并且附加的接口实现方法通过委托形式通知调用。因此使用classcode 可以在不实现接口的前提下动态的给类添加接口实现，而且这些接口实现都可以正常调用。
二、可以通过代理方式生成新类对象，代理方式生成的类对象可以在不重新创建类对象就完成向对象身上附加接口实现。这个功能是通过静态代理模式实现的。
三、可以在新生成类的上面加装AOP特性，并且AOP特性可以控制是否打开来优化生成的字节码。 继承方式--继承方式实现新类，这种生成模式下必须要求先有类型后有对象。生成的新类是继承原有类实现的， 所有附加方法都写到新类中。原始类中的所有方法都被重写并且以super形式调用父类。私有方法不包括重写范畴。 私有方法将不参与AOP功能。在继承模式下保护方法与公共方法参与AOP功能。
代理方式--代理方式实现新类，这种生成模式下可以在已有的对象上附加接口实现而不需要重新创建对象。同时生成的新对象 不破坏原有对象。整个实现方式就是一个静态代理方式实现。注意这种生成方式会取消所有原始类中的构造方法。 取而代之的是生成一个一个参数的构造方法，该参数类型就是基类类型。所有方法调用都使用这个注入的类型对象调用。 同时该中生成方式的私有方法不包括重写范畴。
在代理模式下只有公共方法参与AOP功能，私有方法和受保护的方法因访问权限问题不能参与AOP。

注意：
由于当前版本不支持直接装载rt.jar的类对象因此无法使用rt.jar中的 对象直接做基类类型。这个问题的临时解决方案是开发人员可以编写一个类继承需要的rt.jar中的类比如Object然后将这个类型作为基类使用。导致这个问题的是 由于jvm类装载时jvm.dll首先装载rt.jar，所有rt.jar中的类无法索引到其ClassLoader对象。这个ClassLoader是C语言实现的因此无法获取其ClassLoader类型对象。 同时也就无法通过其ClassLoader获取其class文件。另外当前版本中无法获取已经是动态生成类的字节码数据。因此也无法使用动态类型对象作为基类类型。这些都是 由于无法获取其class字节码数据而致。
