submit系统v3.0的外壳提供程序，该外壳是使用spring作为容器。 在spring的配置文件中并不是所有的bean都是action。如果想使一个bean成为action需要像如下进行配置：
<bean name="mrmiAction" class="org.more.mrmi.JavaScriptSubmitManager">
<meta key="isAction" value="true" /><!-- submit使用 -->
</bean>
spring中的配置说明：
isAction：表示是否为一个action。
isPublicFilter：表示是否为一个全局action过滤器，如果配置了全局过滤器则任何action都将使用这个过滤器。
actionFilters：该属性只有当配置了isAction属性为true之后才会生效，表示这个action所使用的私有过滤器的bean名称集合以逗号分隔。

如果使用spring作为容器提供程序的web部署需要在配置submit3.0时候配置参数buildClass为org.more.submit.casing.spring.WebSpringBuilder。 这个时候configFile参数相当于Spring的配置文件位置，如果配置了Spring的监听器则不需要配置configFile参数。
